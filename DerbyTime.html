DerbyTime!

A .Net 4 Pinewood Derby race scheduler/manager

Download .zip Download .tar.gz View on GitHub

Welcome to DerbyTime!

DerbyTime is a personal project I started in February 2015 on behalf of my son, a Cub Scout, when I was suddenly picked to manage this year's Pinewood Derby for the pack.

Scanning online, decent race management software out there was going to cost too much money, and took away personal control. There was free software available, but it was often highly limited or focused near-exclusively on the elimination methods. I wanted to avoid that because I wanted to keep the kids actively involved in racing for as long as possible.

I decided to write the project using .Net 4, as a decent number of packs may still be using older systems that cannot yet run .Net 4.5.

Project Status (2015-03-08)

This project is still in development. Estimated completion is 2015-03-14:

  • The scheduler library is 100% complete. 3 different round-robin approaches are provided.
  • The EXE stands at ~70%. The basic shell structure is complete, as is the configuration editor and the driver manager, but not the race manager. Pretty much all of the resources are loaded except trophy/ribbon icons for race placings.
  • The MSI (WiX) installer is a stretch goal, and has only just been started.

Project Details

The scheduler library provides 3 different round-robin methods for scheduling a race. All methods can take from 2-6 lanes, and from 2-200 cars (must have at least as many cars as lanes). All methods result in each car racing once per lane per run. All methods allow for randomizing the heats in a given run and the runs themselves. All methods remove the need to use Byes (placeholder cars) to complete run schedules.

  • Basic Lane Rotation -- Generates 1-12 runs. Low optimization - cars will race only those close in the numbering sequence.
  • Chaotic Lane Rotation -- Generates 1-12 runs. Medium optimization - cars will race a semi-random mix of other cars (constraint of 1 race in each lane still applies).
  • Enhanced Lane Rotation -- Generates 1-12 runs, but available counts may differ for different lane/car combinations. High optimization - creates Perfect-N charts when possible (each car races each other car the same number of times) and Partial-Perfect-N when necessary (each car races each other car as close to the same number of times as possible, with a variance of at most 1). If no PN/PPN solution exists, produces a chart similar to Chaotic.

The EXE provides a consolidated viewport through which one can set basic options, add/remove drivers from the roster (and assign numbers), save/load rosters, generate/run the race, and view information about the race (in a separate window, suitable for displaying via projector or additional screen). Winners can be visible at the pack as a whole, as well as by den (if the information was provided at registration).

The MSI is a stretch goal. It will allow for basic install/uninstall, and thus give the program an operating folder (since it creates a few operational files).
